24 /*
25  * A GContainer is a logical grouping for interactors.
26  * The container manages the position and size of the interactors inside it.
27  * This class is very similar to the Java concept of a container, represented
28  * in Swing by the JPanel class.
29  *
30  * A container has a layout that guides how it positions its interactors.
31  * Supported layouts include border (put interactors in the N/S/W/E/Center),
32  * grid (rows and columns of interactors), and flow (interactors go in a
33  * single horizontal or vertical line).
34  *
35  * Containers also use a "box model" similar to the CSS box model with margins
36  * and padding around each interactor, and spacing between neighboring widgets:
37  *
38  * <pre>
39  container
40  * +-------------------+
41  * | margin |
42  * | +---border----+ |
43  * | | padding | |
44  * | | content | |
45  * | | padding | |
46  * | +-------------+ |
47  * | margin |
48  * | |
49  * | spacing |
50  * | |
51  * | margin |
52  * | +---border----+ |
53  * | | padding | |
54  * | | content | |
55  * | | padding | |
56  * | +-------------+ |
57  * | margin |
58  * | ... |
59  * +-------------------+
60  * </pre>
61  */
