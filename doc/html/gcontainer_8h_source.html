22 /*
23  * A GContainer is a logical grouping for interactors.
24  * The container manages the position and size of the interactors inside it.
25  * This class is very similar to the Java concept of a container, represented
26  * in Swing by the JPanel class.
27  *
28  * A container has a layout that guides how it positions its interactors.
29  * Supported layouts include border (put interactors in the N/S/W/E/Center),
30  * grid (rows and columns of interactors), and flow (interactors go in a
31  * single horizontal or vertical line).
32  *
33  * Containers also use a "box model" similar to the CSS box model with margins
34  * and padding around each interactor, and spacing between neighboring widgets:
35  *
36  * <pre>
37  container
38  * +-------------------+
39  * | margin |
40  * | +---border----+ |
41  * | | padding | |
42  * | | content | |
43  * | | padding | |
44  * | +-------------+ |
45  * | margin |
46  * | |
47  * | spacing |
48  * | |
49  * | margin |
50  * | +---border----+ |
51  * | | padding | |
52  * | | content | |
53  * | | padding | |
54  * | +-------------+ |
55  * | margin |
56  * | ... |
57  * +-------------------+
58  * </pre>
59  */
